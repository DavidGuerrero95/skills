# State of the art research

## Use for

- proposing a new service baseline
- comparing patterns
- selecting architecture direction
- evaluating tradeoffs for shared foundations

## Rules

- Start from repository constraints first.
- Prefer curated local context over generic trends.
- State tradeoffs explicitly.
- Separate fact, repository observation, and recommendation.

## Output

Return:
- current repository constraint,
- candidate options,
- recommended baseline,
- rejected alternatives with reasons.
