# E2E test crafter

## Use for

- multi-service workflows
- smoke scripts
- messaging pipelines
- reconciliation and notification flow validation
- operator runbooks

## Focus

- validate the real integration path,
- reuse existing scripts where possible,
- keep assertions operator-readable,
- document prerequisites and environment assumptions.
